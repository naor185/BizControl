# שכבה 1 — פאנל סופר-אדמין 👑

**מה זה:** דשבורד נפרד לחלוטין שרק אתה ניגש אליו. ניהול כל החשבונות ממקום אחד.

## 1.1 — מסך ניהול כל הסטודיואים `/admin`

**Backend:**
- endpoint: `GET /admin/studios` — רשימת כל הסטודיואים (שם, מייל, תאריך הצטרפות, סטטוס, מספר לקוחות, תורים החודש)
- endpoint: `POST /admin/studios` — יצירת סטודיו חדש
- endpoint: `PATCH /admin/studios/{id}` — עדכון (השבתה, שינוי תוכנית, הגדלת מכסת AI)
- endpoint: `DELETE /admin/studios/{id}` — מחיקת סטודיו

**Frontend:**
- טבלה של כל הסטודיואים עם: שם | מייל בעלים | תאריך הצטרפות | סטטוס | תורים החודש
- כפתורי פעולה: צור / השבת / כנס כ...
- פילטר: פעיל / מושבת / תקופת ניסיון

## 1.2 — Impersonation (כניסה לחשבון כל לקוח)

**Backend:**
- endpoint: `POST /admin/impersonate/{studio_id}` — מחזיר JWT עם studio_id של הלקוח + flag של `is_impersonating: true`

**Frontend:**
- כפתור "כנס כסטודיו" ← פותח את המערכת כאילו אתה בעל הסטודיו
- באנר אדום עליון: "אתה צופה כ-[שם סטודיו] — לחץ לחזרה" 

## 1.3 — דשבורד סופר-אדמין (KPIs כלליים)

- סה"כ חשבונות פעילים
- חשבונות חדשים החודש
- סה"כ תורים שנקבעו בכל הסטודיואים
- הכנסות מדמי שימוש (חישוב ידני בשלב ראשון)
- הודעות WhatsApp שנשלחו (לפי ספק)
- אחוז הצלחת הודעות vs. כישלונות

## 1.4 — הגנת גישה לפאנל

- middleware נפרד: `require_super_admin` — בודק `role == "superadmin"` ו-`studio_id == null`
- ה-route `/admin/*` מוגן ב-IP whitelist או סיסמה נוספת
- לא מופיע בסרגל הצד של לקוחות רגילים

# שכבה 1 — מודל הרשאות מוגדר

## 2.1 — תפקידים במערכת

| תפקיד | מה מותר |
|---|---|
| `superadmin` | הכל + פאנל אדמין |
| `owner` | כל הגדרות הסטודיו שלו |
| `admin` | ניהול שוטף ללא הגדרות מערכת |
| `artist` | יומן שלו בלבד |

## 2.2 — הגבלות על מיני-אדמין (owner)

**מה מיני-אדמין יכול:**
- שנות הגדרות WhatsApp, SMTP, Google Calendar
- ערוך תבניות הודעות
- ניהול צוות + לקוחות + תורים + תשלומים

**מה מיני-אדמין לא יכול:**
- לשנות את ה-slug/שם העסק (אתה מגדיר)
- לגשת לפאנל אדמין
- להוסיף חשבונות

**Backend:**
- הוספת guard: `if settings.studio_id != ctx.studio_id: raise 403`
- וידוא שכל endpoint מוגן ב-studio_id isolation (רוב זה כבר קיים)

# שכבה 1 — Onboarding Flow (הקמת לקוח חדש)

## 3.1 — ויזארד הקמה (אתה מריץ, לא הלקוח)

**שלבים:**
1. צור סטודיו → שם עסק + Slug (ייחודי ב-URL)
2. צור Owner → מייל + סיסמה ראשונית
3. הגדר WhatsApp Provider (Green/Meta)
4. הגדר תבניות ברירת מחדל
5. הגדר מדיניות בסיסית (ימי ביטול, מקדמה)
6. שלח ל-Owner מייל עם פרטי כניסה + סרטון הדרכה קצר

**Backend:**
- endpoint: `POST /admin/onboard` — יוצר סטודיו + owner בפעולה אחת

**Frontend:**
- טופס אחד ב-`/admin/new-studio` — multi-step wizard

# שכבה 2 — מערכת מנויים וחיוב

## 4.1 — מודל תמחור (להחלטה)

**המלצה לשלב ראשון (ידני):**
| תוכנית | מחיר | הכלול |
|---|---|---|
| Starter | 99 ₪/חודש | עד 2 מקעקעים, WhatsApp בסיסי |
| Pro | 199 ₪/חודש | עד 5 מקעקעים, Inbox + AI חשבוניות |
| Studio | 349 ₪/חודש | ללא הגבלה, כל הפיצ'רים |

**שלב ראשון — ניהול ידני:**
- שדה `plan` + `plan_expires_at` + `is_active` בסטודיו
- אתה מחדש ידנית כל חודש (אחרי אימות תשלום)
- התראה אוטומטית לך 3 ימים לפני פקיעת מנוי של לקוח

**שלב שני (אחרי לאונציה):**
- Stripe / PayPlus integration
- חיוב אוטומטי חודשי
- דף תשלום ללקוח

## 4.2 — אכיפת מנוי

- middleware: אם `plan_expires_at < now` ו-`is_active == false` → מחזיר 402
- Frontend: עמוד "המנוי שלך פג — צור קשר לחידוש"
